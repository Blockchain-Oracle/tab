B1 · TAB SDK COMPONENT

<PayButton> in the merchant page

The button is the story: IDLE → LOADING → AUTH → PROCESSING → SUCCESS. It disables during LOADING and PROCESSING (double-submit blocked). SUCCESS morphs in place — the checkmark draws in over 450ms, then the label reads “Done”.

B1 — IDLE
First handshake: “Pay $12.00” + lock. Enabled.
B1 — LOADING
Intent fetch + auth init. Disabled, spinner, no interim screen.
B1 — AUTH
Sheet has focus; page and button dimmed behind the scrim.
B1 — PROCESSING
Reassurance state. Disabled; double-submit blocked.
B1 — SUCCESS
Animated: green flood + checkmark draw-in, label morphs to “Done”.
B1 — ERROR
Calm, blame-free. Re-enabled in place; buyer data preserved.
B1 — DISABLED_INSUFFICIENT_FUNDS
Disabled but legible; message is visible and blame-free.

B3 · MAGIC SDK (MOCKED)

6-box OTP

The verified Privy/Dynamic pattern: back arrow, envelope, masked email, six boxes, auto-advance, auto-submit on the 6th digit, resend link. The same component reappears in merchant sign-up and log-in (MA1/MA2).

B3 — OTP_AWAITING_INPUT
First box focused with caret; auto-advance per character.
B3 — OTP_AUTO_SUBMITTING
6th digit entered → submits itself. No confirm button exists.
B3 — OTP_WRONG_CODE
Boxes clear and turn red; retry in place.
B3 — OTP_EXPIRED
A new code is already on its way — no dead end.
B3 — OTP_RATE_LIMITED
Boxes disabled; wait copy with explicit duration.
B3 — OTP_NOT_RECEIVED
After resend: ~30s cooldown before the link re-activates.
B3 — RETURNING_USER_SKIP
Valid session → B2/B3 never appear. LOADING lands directly on the checkout panel while the balance shimmers in.

B6 · TAB SDK

Error / retry panel

The buyer is never blamed and never sees a technical error. Retry happens in place — no reload, data preserved. Every string below passes the vocabulary ban list.

B6 — INSUFFICIENT_BALANCE_ERROR
No retry offered — nothing to retry until the buyer has funds.
B6 — TRANSACTION_FAILED
“You have not been charged” — retry in place.
B6 — NETWORK_ERROR
Connectivity failure during any async call.
B6 — MAGIC_AUTH_FAILED
Terminal auth failure → restart from the email step.
