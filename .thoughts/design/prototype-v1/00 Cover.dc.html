Tab + Leash

Two products, one rail. Tab makes crypto checkout feel like Stripe Link — an email, a 6-digit code, one button. Leash auto-pays x402 paywalls for your own agent — Claude Code, Cursor, any script — from a capped, watched, instantly revocable treasury. It has no LLM of its own. One balance powers both sides of the x402 economy: humans pay via Tab, agents auto-pay via Leash. The same design that wins the demo is the product.

The system

The product’s honest character — crypto made invisible for buyers, clear trustworthy control for operators — drives the aesthetic: warm paper neutrals, light mode, hairline borders, no glow, no gradient-crypto anywhere. Mercury / Ramp / Stripe register throughout.

Resolved designer’s-call items

Per the handback contract these choices are canonical for the prototype.

BUYER
  • Container: bottom sheet at 375px, centered modal on desktop (M1)
  • Magic OTP + Tab checkout share one visual container — sequential views in the same sheet
  • CTA verb: “Pay $12.00” (explicit amount everywhere)
  • Success panel: explicit close, no auto-dismiss; checkmark = CSS/SVG stroke draw, ~500ms
  • Close from B4 returns the button to IDLE, not ERROR; close hidden while processing
  • Returning-user skip lands on B4 with a balance shimmer (OQ-B4)
  • Merchant identity: initial tile + name in the sheet header; amount echoed in header and CTA
MERCHANT
  • Navigation: sidebar; test/live segmented control pinned at its foot on every screen
  • Transaction detail: Stripe-style side panel
  • Transactions list: pagination, not infinite scroll
  • Quickstart: key creation + first test payment auto-detected; other steps manual
  • “Send test webhook” and manual “Resend delivery”: included
  • Go Live modal includes a static pre-flight checklist
  • tokenChanges: structured rows + collapsed raw JSON
  • Payer type column (stretch R-RAIL-2 / OQ-S1: included, flagged) — Human/Agent chips; one agent row in mock data
  • Key permissions: “Full access” / “Sending access”
LEASH WEB
  • Sidebar navigation, routed sections; same shell architecture as Tab (one rail, one family)
  • Receipt detail (W3): side panel over the feed
  • Spend bar palette: green #206B4E → amber #E0A33E → red #C13A40; overage renders a hatched overflow segment
  • Live treatment: pulsing “Live · ~3s polling” chip + newest-row tint (slide-in on prepend)
  • Level 3 confirm: type CANCEL; Level 4: type the agent’s name + filled-red destructive button
  • Connect your agent (replaces the model picker — Leash has no LLM): Leash MCP server recommended, HTTP wrapper as the alternative; key is a one-time reveal (same pattern as Tab’s API keys); status ladder not connected → awaiting first payment → active / error
  • Multi-chain treasury: one unified total over per-network floats — Base primary, Arbitrum, Polygon; background rebalance shown as a “topping up” row
  • Tier-1 badge: in-app sidebar counter only
  • Feed rows carry the settlement network (Base / Arbitrum / Polygon); explorer links resolve per chain — Basescan, Arbiscan, Polygonscan
LEASH MOBILE
  • Navigation: 3-tab bar — Overview / Feed / Controls
  • Revoke container: bottom sheet; Levels 1–2 are toggles with a single confirm
  • Level 3: type CANCEL; Level 4: type the name + hold-to-delete (3s)
  • Paused vs Frozen (OQ-P4): warm-gray “loop halted” vs steel-blue “transactions blocked” + distinct copy
  • Feed rows: USD only; atomic units + network live in the receipt sheet
  • txHash shown short-form in feed rows; tappable
  • Empty states: plain text + small geometric glyph — no illustration
  • P1 shows the unified treasury total plus a compact per-network line (Base · Arb · Pol); receipt sheet carries the CAIP-2 id + per-chain explorer
